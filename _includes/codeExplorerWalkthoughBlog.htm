This blog is going to be about utilizing the Quick Access Tool bar in Windows 10 along with basic Command Line skills. This article is targeting developers that are working on or learning from an existing project, but you may find other uses for it too.  By the end of this tutorial I hope that you can benefit from these configurations to your Windows 10 PC to expedite the learning process as well as the development process.

 

 I am approaching this tutorial in the instance, you are working through a different tutorial with the given file structure as an example.  It would be helpful to navigate to the example directory and then open Visual Studio code from the command prompt to easily switch between all of the files in a given chapter or module. Because this focuses on the use of Visual Studio Code .. navigate to visual studio code website and download the text editor. If you don’t already have Visual Studio Code installed you can find it here…  https://code.visualstudio.com/

 

Also some windows configurations we will be fine tuning related to the file explorer quick access toolbar, which with make navigating to the desired directory more easily.

 

After downloading and installing Visual Studio Code go ahead and open Visual Studio Code by locating it via the Windows Start menu. On initial load you can optionally add extensions that suit your development style. The reason I had you open Visual Studio Code is to make sure that it successfully installed and you make it past the initial dialogs on first load. You can close Visual Studio Code after your first startup. If you leave it open. The command line will open another instance.

 

First step in configuring the Quick Access Toolbar on the File Explorer. Locate the Quick Access Toolbar

Which should look something like this…

 


 

For those that are unfamiliar with this Quick Access Toolbar, these are simple events that are mapped to a single click that are shortcuts to perform specific actions. Hovering over these icons will display a Tooltip that also gives information as to which each button click event does.

 

Moving on to the next step is to then click the File button and which should display a dropdown that looks like this

Right click on the Open command prompt and select Add to Quick Access Toolbar. You should then see the Command prompt added to the quick access toolbar. The next step would be to navigate to the directory where your project is or the practice project.

 

Once you are in the directory in which the project is in click on the newly added Command Line Icon in the Quick Access Toolbar. This should open a command prompt at the given location. The next step is the simplest of them all. Type in the command: “code .”

That is the word “code” as a command followed by a space then a period as the arguments. This should then open the location of the file/folder you are wanting to view/edit. And Viola  there you have easy way to navigate to a lesson or sample project you may be viewing/editing/reviewing/ learning from. In the instance of editing a web page you can quickly edit via Visual Studio Code, and test changes in the web browser of your choice.

Above is the simple tutorial if you are familiar or would like to get more familiar with command line skills here is a table for those that want to go the extra mile and learn a little more …or just a refresher. 

Command

Description

 

dir

List the contents of the current directory

 

cd

Change directory (tip: after typing “cd” start typing a letter of the next path that is in the directory for example if a folder called Documents is within the directory press “d” then Tab to auto complete if the folder/directory exists within your current path.

cd ..

Go up one directory

 

cd [path]

Change directory to a given path. (if file explorer is open you can right click on the path copy and paste into the command prompt to easily CD to a absolute path

exit

Close the window

cls

Clear the screen

 

explorer

Open the file explorer from command line. Similar to the code command add a period to open at the current location.

code

Open Visual studio code. Without passing in any addition arguments this will open up a blank/empty instance of visual studio code. Adding the period opens at the current path.
